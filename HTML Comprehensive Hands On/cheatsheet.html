HTML Cheatsheet

A quick reference for the HTML tags and patterns you will use most often.

Document skeleton

<!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="UTF-8">
	<meta name="viewport" content="width=device-width, initial-scale=1.0">
	<title>Page Title</title>
</head>
<body>
	<!-- visible content -->
</body>
</html>

Core tags

Tag Use
h1 to h6Headings in descending importance.
pParagraph text.
aLinks to pages, files, or sections.
imgEmbedded images with meaningful alt text.
ul, ol, liUnordered and ordered lists.
tableTabular data.
formUser input and submission.

Semantic layout

  • header: Intro area or page banner.
  • nav: Main navigation.
  • main: Unique page content.
  • section: Themed group of content.
  • article: Self-contained content.
  • aside: Supporting or related content.
  • footer: Closing information.

Important attributes

  • href for links.
  • src and alt for images.
  • id for unique anchors and labels.
  • class for reusable styling hooks.
  • name for form fields.
  • aria-label for accessible naming when needed.

Forms checklist

  1. Pair every input with a matching label.
  2. Use the correct type, such as email, password, or number.
  3. Add placeholder text only as a hint, not as the label.
  4. Use button for actions and submission.
  5. Mark required fields with required when appropriate.

Media

  • Use img for pictures with descriptive alt text.
  • Use audio and video for media playback.
  • Use figure and figcaption for grouped media.

Best practices

  • Write one clear h1 per page.
  • Prefer semantic elements over generic div wrappers.
  • Keep indentation consistent.
  • Use readable link text instead of “click here”.
  • Validate your markup before committing.
